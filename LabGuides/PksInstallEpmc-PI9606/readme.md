## Install PKS 1.6 with Enterprise PKS Management Console (EPMC)

Please be sure to complete the requirements in the [Prerequisites](#prerequisites) section above before proceeding.

### Step 1: Download EPMC OVF File & create CA signed Certificate

1.1 Users of this lab will need to use their own access to download the files for EPMC. Please download the EPMC OVA file to control center before proceeding.

1.2 The PKS-Ninja-T1 Lab Environment utilizes Control Center as the local Certificate Authority. Control Center runs Windows 2012R2 Server with Microsoft Certificate Services Enabled. Before deploying epmc, first create a CA signed certificate for EPMC. 

From the control center desktop, click the windows start button and open Windows Powershell from the start menu. Enter the following commands to create a certificate for epmc-01.corp.local:

``bash

import-module C:\hol\ssl\hol-ssl.psm1
New-HostSslCertificate -HOST_SHORTNAME epmc-01a -HOST_IPv4 192.168.110.28

```

On the control center desktop, open file explorer and navigate to `C:\hol\SSL\host\epmc-01a` to find the certificate files generated by the above command. The epmc-01a ssl certificate is saved as `rui.crt`, the public key is `rui.key`, and the root CA certificate is saved as `C:\hol\SSL\CA-Certificate.cer`. Open these 3 files in Notepad++ as you will need to copy the text in a later step.

### Step 3: Clean out old NAT rules andCreate NAT Rules to Connect to the epmc-01a web UI

Navigate to the `Advanced Networking & Security > Networking > NAT` page, ensure the `Logical Router` value is set to `t0-pks` and delete all existing NAT rules
